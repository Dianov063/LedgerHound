# Payment Safety Country Research

This directory defines the country-by-country research used to expand the payment recipient database. Each country file should result in a small, reviewable product change instead of a global list assembled from assumptions.

## Rollout order

1. United States: peer-to-peer payment apps and marketplace micro-scams.
2. United Kingdom, Canada, and Australia.
3. European bank transfer and local payment systems.
4. Country-specific systems such as Pix, UPI, SPEI, GCash, M-Pesa, and PromptPay.

## Research rule

Use official payment-provider, regulator, consumer-protection, and law-enforcement sources. Record the date checked. Platform rules and dispute options can change, so product text must not promise recovery or legal protection.

## Safety rule

LedgerHound publishes a warning about a payment identifier only after independent corroboration. A cluster is not a court finding and must never be presented as proof that a named person committed a crime.

Full payment identifiers, receipts, contact details, and uploaded evidence stay private. Public pages display only masked identifiers and aggregate report information.
